# Impeccable Design Context
# raoul.ro — Raoul Paraschiv personal website

## Design Context

### Users
Game designers, developers, and studios evaluating Raoul as a potential hire or collaborator. They visit from a desktop during work hours, likely after finding a link on social media or a job board. They are visually literate and will immediately recognize lazy or templated design. Secondary audience: peers in the game design community reading the Commonplace Book posts.

### Brand Personality
**Three words**: playful, clean, organized

The first impression should be: "what a creative person" — a genuine moment of pleasant surprise. Not surprise through gimmick or spectacle, but through the confidence of specific choices. The site should feel like it was made by someone who thinks carefully about systems — because it was.

Tone: warm but precise. Opinionated but not self-important. The writing will be analytical; the design should make that feel inviting rather than dry.

### Aesthetic Direction
**Light mode only** (dark mode to be added later without architectural changes).

**Reference**: pippinbarr.com — the single-column, text-forward structure and radical plainness. But where Pippin's site has zero personality by design, raoul.ro should have *authorship* — choices that feel like they were made by a specific person.

**Anti-reference**: generic developer portfolios with hero sections, project cards on a grid, and a "hire me" CTA. Also: anything that looks like it was generated by AI (purple gradients, glassmorphism, cyan-on-dark, rounded-corner icon grids).

**Fonts**:
- Headings: **Bricolage Grotesque** (Google Fonts, variable) — geometric grotesque with optical size axis. Used at confident sizes with tight tracking. Feels designed-for, not defaulted-to.
- Body: **Literata** (Google Fonts, variable) — designed specifically for screen reading. Warm, slightly bracketed, excellent at small sizes. Pairs with Bricolage without competing.

**Colors** (OKLCH):
- Background: `oklch(98.5% 0.004 60)` — warm off-white, subtly tinted toward the red accent hue
- Foreground: `oklch(18% 0.008 60)` — near-black, slightly warm
- Muted text: `oklch(52% 0.008 60)` — warm mid-grey
- Border: `oklch(88% 0.006 60)` — warm light grey
- Accent: `oklch(47% 0.155 25)` — refined version of the original red; used sparingly

**Accent philosophy**: The accent appears only at hover states, the active nav item, inline code, and timestamps. Its rarity is the point — each appearance feels intentional.

### Design Principles

1. **Organized surprise** — the layout is predictable and scannable (readers can navigate without learning the site), but the typographic choices create a moment of "this is someone's site, not anyone's site."

2. **Earned color** — the accent color is a reward for interaction, not ambient decoration. A page at rest should be almost entirely neutral.

3. **System-thinker's clarity** — every element has a single clear job. No element is present to fill space or look designed. This is the same principle Raoul applies to game systems.

4. **Readable first** — this is a writing site. Typography is the product. Measure, line-height, and font size decisions take priority over visual decoration.

5. **Lightweight by conviction** — no CSS frameworks, no JavaScript for decoration, no web fonts beyond the two chosen pairs. Fast loading is a form of respect for the reader's time.
